# NYC Stoop — UX review (round 2)

*Re-reviewed live at nyc-stoop.vercel.app after the fixes shipped. Walked the same three personas, exercising the flows that were broken last time. Compared against the first review (overall 7/10).*

---

## Headline: the three must-fixes all landed

**Plan my night now delivers a night.** Choosing *Tonight* produced a focused **evening** plan — Village Vanguard 10:30pm → Blue Note 12:35am → Birdland — built from just those picks, header reading "3 stops · 2 meals" with no phantom second day. Last time this gave a two-day morning museum crawl. This is the single biggest change.

**Checklist and Full plan now agree.** Both views show the same times for the same stops (10:30pm / 12:35am / 2:55am), in order, with walk/subway legs between. Last time they disagreed and the Checklist order was scrambled.

**"This Week in NYC" is clean.** It now shows real events (Green Market, Celebration) — the "FWC2026" permit code and the block-by-block address dump are gone.

## Also verified fixed

- **Tonight relative dates** — on the Friday view, cards read "Friday · Doors 8pm," not "Tonight."
- **Mood recommendations populate.** "Just chilling · Upper East Side · Drinks" — the exact combo that returned *nothing* last time — now surfaces Burnside (the real UES whiskey bar). Every activity advertises picks; the flow no longer dead-ends empty.
- **Card consistency.** Mood results are now one uniform compact row (curated, imports, restaurants, editorial all match), and tapping any of them opens the same in-app bottom sheet instead of some opening a page and others jumping to Google Maps. The sheet is enriched (description, rating, neighborhood, "Open in Google Maps") and shows a colored initial tile when there's no photo.
- **Map tab** — Map is now the default and on the left, the jarring white top bar is gone (blends with the theme), the filter reads "Classical" (not "Music"), and there's a new "Food" filter + legend entry.
- **"Los Tacos No. 1"** tip renders in full (no more mid-abbreviation cut), and **"From your list" no longer double-lists** venues that are already editorial picks (Don Angie dropped out; list went 25→23).
- **"A night at Carnegie Hall"** replaced the stale "spring season" copy.
- **Warm coral theme** reads vital, not melancholic, and the nav now correctly highlights the tab you're on.
- **"Where to eat"** is back as the first card in "What kind of day?".

## New / remaining issues

### Should-fix

1. **Evening plans mislabel meals.** The 10:30pm–3am jazz plan inserted a **"Lunch"** slot, and **lunch + dinner were the same restaurant** (Corner Bistro twice — the same-spot dedup didn't fire). For a night-out plan, a lunch slot doesn't belong; it should be dinner and maybe a late-night bite, and the two meals shouldn't duplicate.
2. **Day-summary time range is wrong for evening plans.** The header said "8:00pm – 12:00am · 3 stops" while the stops actually run 10:30pm–~5am. The summary isn't using the sequenced clock.
3. **Three same-type venues stacked too late.** "Surprise me" picked three jazz clubs back-to-back ending ~5am. The night-builder should cap to ~1–2 venues, mix categories, or compress the timing so a "night" is realistic.
4. **Mood pick counts over-promise.** The activity step said "Drinks · 5 picks," but the area-filtered result was 1 (Burnside). The count is mood-level, not neighborhood-filtered, so it advertises more than it delivers — the inverse of the old "filter shows 1 but nothing" bug.

### Polish

5. **Raw cuisine id in the sheet.** Burnside's sheet shows "$$ · **bar_tavern** · Upper East Side" — the internal id, not "Bar." And every restaurant uses the food icon/category even when it's a bar.
6. **Thin data still leaves gaps.** Drinks/coffee per-neighborhood often have only 1–2 real entries (e.g., 1 UES bar). The logic now surfaces everything that exists, but reaching a true 5 needs more venues added to the database — as you planned to do manually.
7. **Mood place-picker map is still cool-toned.** The schematic on the *mood* flow still has the old light-blue backdrop; only the Map tab's schematic was warmed.
8. **Photos depend on the deployed key.** Editorial venues now have Wikimedia photos and the no-photo fallback is a clean colored tile, but the 432 imports still load photos live from Google — confirm `VITE_GOOGLE_MAPS_API_KEY` is set in the Vercel build and the domain is in the key's referrer allowlist, or they stay as initial tiles.

## The one thing I'd fix first

**Tune the evening Plan-my-night output.** It now correctly builds a night, but the night it builds is rough: a lunch slot at 11pm, a duplicated restaurant, three jazz sets ending at 5am, and a summary time that doesn't match. Cap it to 1–2 venues + a real dinner (no lunch after dark), de-dupe the meal, and fix the summary range — that turns the marquee feature from "works" into "feels designed."
